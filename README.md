# naer-snap-out

> 一个给 AI 看的 skill，用来在我钻牛角尖的时候打断我、骂醒我、逼我去画画。
>
> 正常人不应该安装这个。你是正常人。关掉这个页面去干正事。

---

## 这是什么？

这是一个通用的 AI Agent skill，支持任何实现 skill 机制的 AI 编程/对话客户端（如 CodeBuddy、Claude Code、Cursor 等）。

我是一个极度善于自我分析的程序员转画师，我的最大毛病是——当我画不出来的时候，我的第一反应不是"再画一张"，而是写一篇三千字的文章去分析"为什么我画不出来"。这个 skill 就是用来对抗这个毛病的。

当我在和 AI 对话时开始长篇讨论方法论、反复自问"是不是应该换种学习方式"、或者开始写"自我诊断书"时，这个 skill 会触发。AI 会：

1. 直接告诉我我在干什么（"你在用写作逃避动笔"）
2. 甩一句我自己曾经写过的话回敬我自己
3. 给我一个最小阻力的动作指令（"现在，打开 Krita，画一个方块"）
4. 然后闭麦，不接受反驳

它的第四步，如果我继续追问，只回复一个字：**"画。"**

## 为什么要把这个公开？

这种行为艺术有两个意义：

1. **公开承诺强化约束**。一旦把这个并发到网上，"被 AI 骂了还继续废话"就变成了一件更丢人的事情。
2. **自我记录**。这是我对自己症结的认识，封装成一个 artifact。将来回头看——要么它发挥了作用，要么它是一个精致但失败的尝试。两种情况都有记录价值。

## 结构

```
naer-snap-out/
├── SKILL.md              # AI 看的指令：何时触发、如何打断、禁止做什么
├── README.md             # 你在看的这个文件，给人看的
└── references/
    └── quotes.md         # 60+ 条我自己写过的语录，AI 用来精准打击我
```

## 安装

把整个 `naer-snap-out/` 目录放到 AI 客户端的 skills 目录下（具体路径取决于你使用的客户端，一般为 `~/.config/<client>/skills/naer-snap-out/` 或项目下的 `.codebuddy/skills/naer-snap-out/`）。

## 前提条件

只看这个仓库没有任何用处。它需要：
1. 一个支持 skill 机制的 AI 客户端
2. 你恰好是那尔本人（因为触发条件基于那尔特有的思维模式）
3. 你恰好今晚又在写方法论而不是画画

第三条出现频率高得令人羞愧。
